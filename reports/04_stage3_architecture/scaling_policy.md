# Политика масштабирования

## Модель региональности
- Базовая модель: один домен + локальные страницы под Махачкалу и Дагестан.
- Не создавать массовые doorway-страницы под каждый район без уникального интента.

## Правила создания новых URL
1. Есть подтвержденный спрос (ключи/лиды/вопросы sales).
2. Есть уникальный контент: кейсы, цены, условия, локальные особенности.
3. Есть хотя бы 3 внутренних ссылки на новый URL.
4. Есть понятный CTA и связь с существующим коконом.

## Анти-дубли
- Canonical: self-referencing для indexable страниц.
- Фильтры/параметры закрываются от индексации, если не несут отдельного интента.
- Единый стандарт ЧПУ в латинице.
